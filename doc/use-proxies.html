How to use proxy servers

Portecle may access the network when fetching certificates or CRL's, or when examining an SSL/TLS connection. If your environment requires using a nontransparent proxy server to access these destinations, Portecle needs to be configured to use it.

Portecle does not implement any proxy support itself, but the standard Java proxy mechanisms are available for use with it; refer to the Java Networking and Proxies and Networking Properties documents for details on configuring HTTP, HTTPS, SOCKS, and system default proxy details using system properties. Add the mentioned properties as arguments to Java (not Portecle), i.e. before portecle.jar or FPortecle, to the command used to start Portecle. For example: java -Djava.net.useSystemProxies=true -jar portecle.jar

Note that the examine SSL/TLS connection functionality does not use HTTPS even when checking the connection to an HTTPS service. Therefore the HTTPS proxy settings have no effect on it, but the SOCKS proxy applies. The SOCKS proxy also applies to other network accesses, see the above mentioned documents for more details.
